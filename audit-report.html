Compost Mill — Digestion Pipeline

21 files · Entry: bin/liminal compost → CompostMill

1
bin/liminal : compost subcommand
CLI routing: add, digest, soup, seeds, status

Parses liminal compost <action> arguments. Actions: add (files to heap), digest (run pipeline), soup start/stop/status, seeds list, status (mill state).

↓
2
CompostMill : add(paths)
Copies files/directories to heap directory

CompostHeap.addFile() / addDirectory() copies input files to the heap staging area. HeapMonitor watches heap size and auto-triggers digest when >80% capacity.

↓
3
Extraction Pipeline (parallel)
MetadataExtractor · RawByteProcessor · SemanticExtractor

MetadataExtractor: file type, LOC, language detection, SHA256 hash, timestamps.
RawByteProcessor: header/tail hex (4KB each), hex chunks (256-char blocks), base64 for small files.
SemanticExtractor: text summary, code semantics (via DNAExtractor), image description (via LLM vision). Internal cache for deduplication.

↓
4
CompostShredder : shredAll()
Decomposes extraction results into CompostFragment[]

Layer-specific shredding: semantic (paragraph/heading split), code (function/class split, 1024-byte threshold), metadata (each field = fragment), raw bytes (each 256-char chunk = fragment). Each fragment gets: id, source, domain, layer, content, metadata, tags.

↓
5
CollisionEngine : runAll(fragments)
7 collision strategies → cross-domain LLM merge

Strategies: timestamp (within 1hr, different domains), size (similar bytes, different types), metadata (shared values), hash (SHA256 prefix similarity), tag (shared tags, unrelated domains), domain opposites (most different), random (10% sampling). Pairs merged via LLM prompt.

↓
6
FragmentScorer : score(fragment)
5-dim heuristic (60%) + optional LLM (40%) → 0-10

Dimensions: novelty (unique word ratio), density (ideas/char), crossDomain (distinct source domains), metadataRarity (unusual values), connectionStrength (cross-domain links). shouldPromote() checks against configurable threshold.

↓
7
SeedBank : add() → DigestGenerator : save()
Promotes top fragments → seeds → digest markdown

SeedBank: persists to seeds.json + latest/*.md. Tracks usage (useCount, usedBy). Prunes old unused seeds by retention days.
DigestGenerator: saves digest/YYYY-MM-DD.md with sections: Heap Stats, Seeds Promoted, Soup Highlights, Domain Heatmap. Optional LLM synthesis for narrative.

↓
8
CompostSoup : cycle() / run()
Continuous evolutionary loop (optional)

Picks 2 random cross-domain fragments → LLM merge → score offspring → replace worst in population if better → promote to SeedBank if above threshold. Uses NoveltyArchive for tracking, SoupStateManager for persistence. Supports AbortSignal for clean shutdown.

↓
9
Integration Bridges
FragmentArchiveBridge · PromptLibraryBridge · SwarmBridge

FragmentArchiveBridge: syncs compost seeds → FragmentArchive (scavenger).
PromptLibraryBridge: registers 4 compost prompts (collision-merge, offspring-scoring, digest-narrative, seed-extraction).
SwarmBridge: provides compost seeds as prompt seeds for swarm generation.
