🛡️ Guardrail Architecture

Two complementary guardrail systems protect the codebase:

M1-M18 Guardrails

Traditional milestone-based guardrails for specific quality checks.

  • M1-M11: Complete
  • M12-M18: Planned

DGF Framework

3-phase Deterministic Guardrails Framework with runtime protection.

  • Phase 1: Foundation
  • Phase 2: Validation & Remediation
  • Phase 3: Evolution

🔄 Data Flow

How a generation request flows through the system:

User Input
    ↓
┌─────────────────────────────────────┐
│  NaturalInterface / IntentRouter    │
│  Detects command vs chat vs agent   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  GeneratorRegistry.route()          │
│  Selects appropriate generator      │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  Ralph Loop (iterative refinement)  │
│  Generate → Evaluate → Accumulate   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  DGF Guardrails (3-phase)           │
│  Validate → Remediate → Learn       │
└─────────────────────────────────────┘
    ↓
Output (code, visualization, or chat)
