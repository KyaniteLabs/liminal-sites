💬 Natural Language Interface

Just type naturally—no command prefixes needed. Liminal's NaturalInterface routes your input intelligently:

You Type Mode What Happens
"Fix the Tone.js validation" Agent LLM plans and executes code changes
"What's the status?" Command Shows harness status
"Tell me about p5.js" Chat Conversational response with SOUL personality

🤖 LLM Mode Agent

Autonomous agent that plans its own steps, reflects on results, and adapts its strategy.

Capabilities

  • Planning: LLM decides which tools to call based on task description
  • Reflection: Analyzes tool results and adjusts approach
  • Multi-turn: Maintains conversation context across steps
  • Recovery: Attempts to fix errors automatically (up to 3 retries)
  • Rollback: Auto-restores backups on failure

Bubble Tea TUI (Next Generation)

A new operator-grade terminal UI built with Bubble Tea (Go) is now active alongside the Ink-based TUI. Key principles:

  • Pane-first architecture: committed history, active response, and status/trust panes
  • Explicit modes: Chat, Inspect, Action, Confirm
  • Tools-first coding harness: non-creative input routes into the tool-using harness/runtime lane by default
  • Confirmation-first mutation: no state changes without explicit operator approval
  • Active-response streaming: responses stream into a dedicated pane, not into transcript history
  • Trust/provenance labels: provider, model, and trust-level badges always visible
  • Generated code untrusted by default: visual badge and behavioral guard

MVP Active Connected to the TS bridge via HTTP + SSE, with bounded self-improvement packets, focus gating, resume safety, and active provider/model selection from runtime config or env overrides.
