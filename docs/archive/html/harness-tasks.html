🔧 Harness Tasks

Self-improvement tasks for the Meta-Harness

What are Harness Tasks?

Harness tasks are JSON files that define code changes for the Meta-Harness to execute. They enable the system to fix itself, add features, and improve over time.

💡 Philosophy: Generators are "dumb" (no self-improvement) but ARE improved by harness fixes. The harness observes → detects patterns → fixes the system.

📄 Task File Format

Tasks follow a simple JSON structure:

{
  "id": "M1",
  "title": "Fix Tone.js Validation Gate",
  "description": "Tone.js validation only fires on domain 'unknown'...",
  "targetFile": "src/core/CodeValidator.ts",
  "search": "    // Check 6: Tone.js API validation...",
  "replace": "    // Check 6: Tone.js API validation...",
  "verifyCommand": "npm run build",
  "risk": "low",
  "complexity": "low",
  "approved": true
}

Field Reference

Field Required Description
id✅Unique task identifier
title✅Short descriptive title
description✅Detailed explanation
targetFile✅File to modify
search✅Exact string to find
replace✅Replacement string
verifyCommand❌Command to verify fix
risk❌low/medium/high
complexity❌low/medium/high
approved✅Must be true to execute

▶️ Executing Tasks

Via TUI

$ liminal tui

# Inside TUI:
> /run M1
✅ Task M1: SUCCESS

Via Natural Language

> Fix the Tone.js validation
→ Detects agent intent
→ LLM plans steps
→ Executes fix
✅ Task completed

Safety Features

  • Build Verification: Runs npm run build after changes
  • Auto-Rollback: Restores backup if build fails
  • Rate Limiting: 5 LLM calls/min, 10 writes/min
  • Path Validation: Only modifies src/, test/, docs/, scripts/

✏️ Creating Custom Tasks

Step 1: Create JSON File

// harness-tasks/MY-TASK.json
{
  "id": "MY-TASK",
  "title": "Add new feature",
  "description": "Description of what this does",
  "targetFile": "src/my/file.ts",
  "search": "// OLD CODE HERE",
  "replace": "// NEW CODE HERE",
  "verifyCommand": "npm run build",
  "risk": "low",
  "complexity": "low",
  "approved": true
}

Step 2: Test the Search

Verify the search string matches exactly in the target file:

# Use grep to verify
$ grep -n "search string" src/my/file.ts

Step 3: Execute

$ liminal tui
> /run MY-TASK
