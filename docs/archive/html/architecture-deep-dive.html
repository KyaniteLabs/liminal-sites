🔬 Architecture Deep Dive

Technical details of LLMModeAgent, NaturalInterface, and the Tool System

🤖 LLMModeAgent

The autonomous agent that plans and executes tasks with reflection.

Architecture Flow

flowchart TD
    A[User Input] --> B[NaturalInterface]
    B --> C{Intent Detection}
    C -->|Agent Pattern| D[LLMModeAgent]
    C -->|Command| E[Direct Execution]
    C -->|Chat| F[Chat Mode]
    
    D --> G[Create LLMTask]
    G --> H[LLM Plans Tool Calls]
    H --> I[Execute Tool]
    I --> J{Success?}
    J -->|Yes| K[Run Build]
    J -->|No| L[Reflection]
    L --> H
    K --> M{Build Pass?}
    M -->|Yes| N[Task Complete]
    M -->|No| O[Rollback]
    O --> P[Report Failure]

Conversation Flow

// 1. Initialize with system prompt
system: SOUL.md content

// 2. Add task description
user: "Fix the Tone.js validation to include music domain"

// 3. LLM responds with plan
assistant: {
  "thought": "I need to read CodeValidator.ts",
  "tool": "readFile",
  "params": { "path": "src/core/CodeValidator.ts" },
  "expectedResult": "See validation logic"
}

// 4. Execute tool, get result
tool: { success: true, data: "...file contents..." }

// 5. LLM plans next step
assistant: {
  "thought": "Found the issue, will apply fix",
  "tool": "applyEdit",
  "params": { ... },
  "expectedResult": "Update gating condition"
}

// 6. Continue until complete or max steps

💬 NaturalInterface

Routes user input to appropriate handler without prefixes.

Intent Detection Patterns

Type Patterns Examples
Agent fix, add, change, implement "Fix the validation"
Command status, tasks, run, preview "What's the status?"
Chat Questions, greetings, discussion "Tell me about p5.js"

Routing Logic

function processInput(input):
  // 1. Check for explicit slash commands
  if input starts with "/":
    return handleSlashCommand(input)
  
  // 2. Match command patterns
  commandMatch = matchCommand(input)
  if commandMatch:
    return executeCommand(commandMatch)
  
  // 3. Check for agent patterns
  if isAgentRequest(input):
    return handleAgentRequest(input)
  
  // 4. Default to chat
  return handleChat(input)

🔧 Tool System

Seven tools available to the LLMModeAgent:

Tool Reference

Tool File Rate Limit
readFile src/harness/tools/ReadFileTool.ts 10 reads/min
writeFile src/harness/tools/WriteFileTool.ts 10 writes/min
applyEdit src/harness/tools/ApplyEditTool.ts 10 writes/min
runBuild src/harness/tools/RunBuildTool.ts 12 builds/min
runTests src/harness/tools/RunTestsTool.ts 12 tests/min
createBackup src/harness/tools/BackupTools.ts 10 writes/min
restoreBackup src/harness/tools/BackupTools.ts 10 writes/min

Safety Features

  • Path Validation: Only allows src/, test/, docs/, scripts/
  • Auto-Backup: Creates backup before any edit
  • Auto-Rollback: Restores backup if build fails
  • Rate Limiting: Prevents abuse
  • Max Steps: 15 steps default (configurable)
