🎨 Generators ("Dumb" by Design)
NO SELF-IMPROVEMENT
⚠️ Design Principle

Generators are intentionally "dumb" - they generate code but don't self-improve. The harness observes failures and fixes the system around them. This separation allows the harness to improve without modifying working generators.

🔌 Wiring Summary
CRITICAL

Recently Fixed (Today)

  • ✅ RalphLoop → Meta-Harness: Now calls onGenerationComplete() after each iteration and on validation failure
  • ✅ run() → Meta-Harness: Error handling reports to harness
  • ✅ E2E Tests → Meta-Harness: Tests report success/failure to harness in finally block
  • ✅ Harness LLM Config: Added harness-specific parameters (lower temp for code fixes)

Still Not Wired (By Design)

  • ⚠️ Generators → Guardrails: Generators don't use SafetyGuardrails - this is intentional. The harness observes failures and fixes the system, not the generators.
  • ⚠️ HarnessUpdater → Auto-Execute: Creates tasks but doesn't auto-run them. Requires manual approval via TUI.

Data Flow

User Prompt
    ↓
RalphLoop (NOW REPORTS FAILURES TO HARNESS)
    ↓
Generators (DUMB - just generate, no self-improvement)
    ↓
Code Output ← Validated ← Scored
    ↓
Meta-Harness (OBSERVES, DOES NOT MODIFY GENERATORS)
    ├── FailureLogger (logs failures)
    ├── PatternDetector (detects patterns)
    ├── HarnessAgent (applies fixes to SYSTEM, not generators)
    └── Task System (M1, M4, M6, M7, M8)
