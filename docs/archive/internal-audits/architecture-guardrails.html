🛡️ GUARDRAIL ARCHITECTURE

Five Types of Quality Gates for Creative Code Generation
⚠️ Critical Architecture Understanding

Quality control isn't just "validation." There are five distinct types of guardrails, each catching different failure modes. Missing any type creates holes in quality assurance.

🔧
1. CODE QUALITY
Universal Structure & Syntax
Structure Checks
  • Balanced braces, parens, brackets
  • Required functions exist (setup, draw)
  • Valid syntax for language (JS/GLSL)
  • No incomplete functions
  • Proper indentation (basic)
Domain Detection
  • Detect p5.js vs GLSL vs Three.js
  • Route to correct validator
  • Apply domain-specific rules
Implementation
src/core/CodeValidator.ts Exists
👻
2. ANTI-HALLUCINATION
Real APIs Only
API Validation
  • Check p5.js functions exist
  • Verify Tone.js API calls
  • Validate Three.js classes
  • No made-up methods
  • Correct parameter counts
Known Hallucinations
  • Tone.js: Non-existent synth types
  • p5: Fake draw modes
  • GLSL: Wrong built-in functions
  • Misspelled API names
Implementation
src/core/CodeValidator.ts (validateToneJS) Partial
🗑️
3. ANTI-SLOP
Reject Low-Effort Output
Generic Pattern Detection
  • "Blue circle on gray background"
  • Random particle soup
  • Copy-paste noise patterns
  • No clear concept or intent
  • Low complexity (lines < 50)
Effort Indicators
  • Unique visual concepts
  • Coherent color palette
  • Thoughtful composition
  • Creative technique usage
  • Not "student project" quality
Implementation
src/core/CreativeEvaluator.ts (emergence, interestingness) Partial
🎨
4. AESTHETIC QUALITY
Domain-Specific Beauty
Visual (p5/Shader/Three)
  • Color harmony
  • Composition balance
  • Visual complexity
  • Motion quality (if animated)
  • Professional polish
Audio (Tone/Strudel)
  • Frequency balance
  • Musical structure
  • Sound design quality
  • No clipping/distortion
  • Pleasing timbre
Implementation
src/core/CreativeEvaluator.ts (creativeScore) Exists
🔒
5. SAFETY
Security & Resource Limits
Security
  • No eval() or new Function()
  • No child_process
  • No fetch to unknown domains
  • No localStorage/cookie abuse
  • Safe CSS (no expression)
Resource Limits
  • Infinite loop detection
  • Memory usage limits
  • CPU throttling
  • Canvas size limits
  • Audio context limits
Implementation
src/core/SafetyGuardrails.ts + src/security/ Exists

📋 Real Examples

🗑️ Anti-Slop Example

❌ SLOP (Reject)
function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(200);
  fill(0, 0, 255);
  ellipse(200, 200, 100, 100); // Just a blue circle
}

Zero creativity. No animation. No concept. "Student project" quality.

✅ NOT SLOP (Accept)
function setup() {
  createCanvas(400, 400);
  colorMode(HSB);
  noStroke();
}

function draw() {
  background(0);
  // Flowing organic blobs with sin waves
  for (let i = 0; i < 10; i++) {
    let x = noise(i * 0.5, frameCount * 0.01) * width;
    let y = noise(i * 0.5 + 100, frameCount * 0.01) * height;
    let size = sin(frameCount * 0.05 + i) * 50 + 60;
    fill((frameCount + i * 30) % 360, 70, 90);
    ellipse(x, y, size);
  }
}

Clear concept (flowing blobs), animation, color harmony, creative technique.

👻 Anti-Hallucination Example

❌ HALLUCINATION (Reject)
// Fake Tone.js API
const synth = new Tone.MegaSynth(); // Doesn't exist!
synth.playNote("C4"); // Wrong method name

// Fake p5 function
setPixelColor(10, 10, 255); // Doesn't exist

LLM invented "MegaSynth" and "setPixelColor". These don't exist in the libraries.

✅ REAL API (Accept)
// Real Tone.js API
const synth = new Tone.PolySynth(Tone.Synth).toDestination();
synth.triggerAttackRelease("C4", "8n");

// Real p5 function
set(10, 10, color(255)); // Actual p5 pixel function

Uses actual documented APIs. M1 task fixes Tone.js validation to catch this!

⚠️ Current Gaps in Guardrails

Anti-Slop: WEAK

CreativeEvaluator has "emergence" and "interestingness" but doesn't explicitly reject "blue circle on gray background" type outputs.

Anti-Hallucination: PARTIAL

Tone.js validation exists but only fires on 'unknown' domain (M1 fixes this!). p5.js hallucination checking is minimal.

Aesthetic: PER-DOMAIN NEEDED

Visual evaluation exists but audio aesthetic evaluation is minimal. Each generator needs domain-specific aesthetic checks.
