## Core Narrative Arc: The Honest Loop

```
WOW → WAIT → WHY → HOW → WHAT'S NEXT
```

| Phase | Emotion | Content Type |
|-------|---------|-------------|
| **WOW** | Awe | 15-60s clips of generated art/music |
| **WAIT** | Skepticism | "But does it actually work?" — show the dogfood dashboard |
| **WHY** | Curiosity | The meta-harness: why it fails, how it learns |
| **HOW** | Understanding | Technical deep-dives: model comparison, prompt engineering |
| **WHAT'S NEXT** | Anticipation | Roadmap, community challenges, live generation |

**Never skip WAIT.** The failures are your credibility. The landing page says "0.77 avg score" — lean into that.

## Content Pillars

### Pillar 1: "It Actually Runs" (Proof)
Show generated code executing. Not screenshots of code — the *result*. Fast cuts between:
- Prompt text appearing
- Terminal running
- Browser opening
- Art appearing

**Best for:** TikTok/Reels/Shorts (15-30s)

### Pillar 2: "The Hall of Shame" (Honesty)
Show failures. Broken shaders. Tone.js that sounds like a dial-up modem. Video generation that outputs a blank frame. Then show the meta-harness detecting the pattern and fixing it.

**Best for:** Twitter threads, YouTube shorts, LinkedIn carousels

### Pillar 3: "Model Hunger Games" (Comparison)
Same prompt, 5 models, wildly different results. Gemma vs Phi-4 vs Qwen vs Granite-350M. Show pass/fail rates, generation time, code size.

**Best for:** Twitter threads, blog posts, YouTube

### Pillar 4: "The Meta-Harness Eats Its Young" (Technical)
How the system observes failures, detects patterns, and adapts prompts/routing without human intervention.

**Best for:** Technical Twitter, Hacker News posts, blog deep-dives

### Pillar 5: "Prompt → Anything" (Creative)
One sentence becomes a GLSL shader, a generative music loop, a 3D scene, a video composition. Show the breadth.

**Best for:** Visual-first platforms (TikTok, Instagram, Pinterest)

## SCRIPT 6: "Live Generation" (Twitter Spaces / Twitch / YouTube Live)

**Purpose:** Real-time interaction. Prove it's not pre-recorded.
**Format:** Live stream, 30-60 minutes

---

**Setup:**
- Screen share: terminal + browser side by side
- Chat visible
- Background audio: ambient Strudel loop

**Flow:**
1. **Intro (5 min):** "I'm going to generate creative code from your prompts live. No editing. If it fails, you see it fail."
2. **Prompt 1 (10 min):** Pick a prompt from chat. Generate. Show score. Discuss.
3. **Prompt 2 (10 min):** Pick a harder prompt. Maybe it fails. Debug live.
4. **The Comparison (15 min):** Run same prompt through 2-3 models. Compare live.
5. **Q&A (15 min):** Answer technical questions. Show code if people ask.
6. **Outro (5 min):** "Thanks for the prompts. All outputs will be added to the gallery with your username."

**Pro tip:** When something fails, that's the best content. Don't rush to fix it. Explain *why* it failed, check the harness logs, show the pattern detection. That's what technical audiences crave.

## Engagement Tactics

### The "Score Reveal" Hook
Never post just the output. Always include the score.
- "Score: 0.91 — this one surprised me"
- "Score: 0.34 — the prompt was too abstract"
- "Score: 0.00 — it crashed the browser tab"

This creates suspense and shows you're not filtering.

### The "Reply and I'll Generate" Challenge
Drop this in replies to your own threads. It drives engagement and gives you content for the next day.

### The "Model Bet"
"I bet $5 that Granite-350M can't generate a convincing ocean scene. Let's find out."
People love stakes, even fake ones.

### The "Failure Friday"
Weekly post showing the 3 worst failures of the week. Normalize failure. Build trust.

## What Makes This Work

1. **Honesty is differentiation.** Everyone else cherry-picks. You literally built "no cherry-picking" into the product.

2. **Data is content.** The dogfood pipeline generates infinite content. Pass rates, model comparisons, failure patterns — it's all storytelling material.

3. **Visuals are shareable.** p5.js and GLSL outputs are inherently beautiful. People share art.

4. **Failure is engaging.** Watching something break, then watching the system learn why, is more compelling than watching perfection.

5. **You're both roles.** The creative coder who wants beautiful outputs AND the engineer who built the evaluation pipeline. That duality is rare and interesting.
